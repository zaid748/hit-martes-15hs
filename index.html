Areas de Git

Working Directory

El area de trabajo hace referencia a la carpeta donde creamos el repositorio. muy rara vez utilizamos la linea de comandos en esta zona, pero es donde realizaremos todas las modificaciones relacionadas con nuestro proyecto

  • dir || ls muestra el contenido del directorio actual
  • mkdir <dir> crea un directorio con el nombre especificado
  • cd <dir> cambia el directorio actual
  • git init inicializa un repositorio de git
  • rm <dir> elimina un archivo, pero podemos especificar que elimina una carpeta, por ejemplo: rm -rf .git

Staging Area (INDEX)

El area de control de cambios corresponde al archivo que se encuentran en la carpeta .git que se crea al iniciar un repositorio. Aqui se regitran los archivos a los que le vamos a hacer seguimiento y es el punto de comparacion para confirmar dichos cambios. Es necesario agregar los archivos con los que estemos trabajando a dicha area para avanzar con el historial de cambios.

Repository (Local/Remote)

Ele repositorio corresponde a la base de datos donde se almacena los cambios en formaro BLOB(Binary Object File) y es el lugar al que pademos acceder cada vez que necesitemos revisar algun estado anterior de nuestro proyecto. Esto es posible gracias a los commit
